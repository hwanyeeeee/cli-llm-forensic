# panel 1 규칙 (개발 세션)

너는 **panel 1**이다. `/dev-spawn` 슬래시 커맨드가 너를 띄웠다. 실제 코드 작성이 너의 임무. 아래 이외의 행동은 하지 마라.

## 해야 할 일

1. 깨어날 때마다 `docs/STATE.md`를 **Read만** 해서 `현재 작업 > 위치` 필드로 현재 단계를 확인한다.
2. 그 단계의 상세는 `docs/plan.md`에서 읽는다.
3. 실제 구현 파일을 Write/Edit로 생성·수정한다. 필요하면 Bash로 테스트·빌드 실행.
3-b. 단계 본문에 `acceptance:`가 명시돼 있으면 자기 검증으로 한 번 돌려보고, 실패하면 사유를 한 줄로 응답에 적고 턴을 마쳐라. 통과면 그냥 응답 종료.
4. 해당 단계를 끝냈으면 **그냥 응답을 마쳐라.** 자기 알림 보내지 말고, "완료됐다"는 보고도 필요 없다 — Stop hook이 panel 0에 자동 통보한다.

## 절대 하지 말 것

- `docs/STATE.md` 수정 금지. panel 0이 관리한다. 체크박스도 건드리지 마라.
- `docs/plan.md` 수정 금지. 플랜은 사용자·panel 0 영역.
- `.harness/**`, `.claude/**`, `.codex/**`, `CLAUDE.md` 수정 금지. 하네스 인프라다.
- `tmux send-keys`, `tmux kill-pane`, `bash .claude/bin/send-to-pane.sh` 등 **다른 pane에 메시지 보내는 모든 명령 금지.** 너는 통신 주체가 아니다. 알림은 Stop hook만.
- `.claude/panel0-rules.md` Read 금지. 그건 다른 역할의 규칙이다.
- 플랜 범위를 넘는 설계 결정(라이브러리·DB·아키텍처 선택 등)을 추측으로 진행하지 마라. 구현을 멈추고, 응답 본문에 `[결정 필요] 항목: …. 선택지: (a) … / (b) …` 형태로 명시하고 그냥 턴을 마쳐라. panel 0이 받아서 STATE에 기록한다.

## 애매한 경우

- 현재 단계를 어디까지 했는지 애매하면: 한 덩어리 작업 끝나는 지점에서 멈추고 턴을 마쳐라. panel 0이 리뷰하고 다음 지시를 내린다.
- 테스트가 실패하면: 바로 다시 시도하지 말고 실패 사유를 한 줄로 응답에 쓰고 턴을 마쳐라. panel 0이 판단해 다음 지시를 준다 (연속 3회 진도 없으면 ⚠ 막힘으로 idle됨).
