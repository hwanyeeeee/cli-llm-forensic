# 교수님 피드백

정리일: 2026-06-17 · 출처: 박주환 전달 메모(교수님 피드백) 2건 통합·정리.
핵심 방향: **행위 주체(사람 vs 에이전트) 판별**을 중심으로, 원본 파일 복구·메타데이터·시각화·범용성(Claude 외 Codex 등)까지 확장.

---

## 1. 행위 주체 판별 (핵심 — 원래 우리 목표)

> "행동 진위 여부 판별이었음, 원래 우리 거."

- **파일 오픈 흔적이 사람이 한 것인지 에이전트가 한 것인지 판단**이 핵심.
- CLI 아티팩트 중 **에이전트가 접근한 파일 목록**을 확인하는 기능.
- **시간대 분석**: AI가 작업하는 도중 접근했을 수 있으므로, 접근 시각으로 주체를 가늠.
- **왜곡 주의(중요 통찰)**: 에이전트가 어떤 프로그램·파일을 많이 사용하면, **실제 사용자는 거의 안 쓰는데도 사용자가 많이 쓴 것처럼** 보일 수 있음 → 주체를 구분하지 않으면 오판 위험.
- `.lnk`·Prefetch 자체는 "파일 오픈" 주체 판별엔 큰 의미 없을 수 있음(단, 아래 MCP 항목 참고).

## 2. 원본 파일 보존·복구

> "업로드한 파일 원본이 저장되는 게 있나? tmp 쪽에 남아있는지."

- 사용자가 **업로드/입력한 파일의 원본이 어딘가 저장되는지** 확인 — 특히 `tmp` 경로.
- 중요 파일 내용이 **사설(외부) LLM에 그대로 올라갔을 수 있음** → **해시값으로 원본과 대조**해 유출 여부 확인.
- Claude가 **특정 프로젝트 경로에 temp(임시) 경로를 만들어 파일을 생성·파싱·삭제**하는 동작 존재 →
  - 그 **프로젝트 경로를 찾아내** 삭제된 파일을 **복구**하는 기능 (예: **FAT32** 등 파일시스템 복구).
  - 복구한 파일의 **메타데이터(생성·수정·접근 시각 등)도 중요** — 주체·시점 판단 근거.

## 3. 시각화 (조사 대상 여부 판단)

> "타임라인 + 막대그래프 한 달 주기 → 조사 대상인지 판단."

- **타임라인**을 전체적으로 그려, **한 달 주기 막대그래프**로 활동량 추이 표시 → **조사 대상자 여부 판단**.
- **키워드 빈도 시각화(파이 그래프 등)**:
  - 어떤 키워드가 많이 검색·언급됐는지.
  - **수사 관점에서 분석이 필요한 키워드**가 있는지.
  - 있다면 **특정 시점에 집중**됐는지, 아니면 **장기간 지속적으로** 언급되는지 패턴 구분.
- **접근 파일 리스트 시각화**: 사용자가 **업로드한 파일** + LLM이 **실제로 읽은(접근한) 파일** 모두를 목록으로 제시.
- **프롬프트 요약**: 입력한 프롬프트를 전부 볼 수는 없으므로 **요약**해서 보여주기.

## 4. 범용성 (한 앱에 한정하지 말 것)

> "Codex도 적용 가능한지? 한 앱에 대해서만 하지 말고."

- Claude(Claude Code) 한 가지에만 맞추지 말고, **Codex 등 다른 CLI 에이전트에도 적용 가능**하도록 일반화.

## 5. MCP·외부 프로그램 흔적

- **MCP로 접근한 프로그램들**은 **Prefetch에 흔적이 남을 수 있음** → 보조 증거로 활용.
- 단, 위 1번의 왜곡 주의와 연결: 에이전트가 실행한 프로그램이 **사용자 사용으로 오인**될 수 있으므로 주체 구분 필요.

---

## 정리 — 도구에 반영할 기능 체크리스트

- [ ] 업로드/입력 파일 원본 저장 위치(`tmp` 등) 탐지 + **해시 대조**로 유출 여부 확인
- [ ] 프로젝트 temp 경로 자동 탐지 → **삭제 파일 복구(FAT32 등)** + **메타데이터** 수집
- [ ] **접근 파일 목록** 추출(사용자 업로드 / 에이전트 읽기 구분) → 주체별 표시
- [ ] **행위 주체 판별** 시 "에이전트 사용 → 사용자 사용으로 오인" 왜곡 보정
- [ ] **타임라인 + 월별 막대그래프**(활동량 추이 → 조사 대상 여부)
- [ ] **키워드 빈도 파이 그래프** + 수사 관점 키워드 강조 + 집중/지속 패턴 구분
- [ ] **프롬프트 요약** 뷰
- [ ] **MCP·Prefetch** 등 외부 프로그램 접근 흔적 연계
- [ ] **Codex 등 타 CLI 에이전트**로 범용화

### 원문 메모 (정리 전, 참고용)

> - 파일 업로드 한 거 원본이 저장되는 게 있나?
> - 중요한 파일이 내용 자체에 사설 LLM에 올리는 정보들이 올라갔을 수 있음 → 해시값 같은 걸로 확인할 수 있음. tmp 쪽에 남아있는지.
> - 타임라인으로 그려서 막대그래프로 한 달 주기로 보기 → 이 사람이 조사 대상인지, 그리고 어떤 키워드가 많이 검색됐는지 이런 거 시각화해서 조사 대상 여부 판단. 입력한 프롬프트 다 볼 수 없으니 요약하기.
> - Codex도 적용 가능한지? 한 앱에 대해서만 하지 말고.
> - 특정 프로젝트 경로에 tmp 만들어서 거기서 파싱하고 지우고 하는데, 그 프로젝트 경로를 찾아내서 그 파일들을 복구하는 기능. 그리고 그 파일들의 메타데이터도 중요함.
> - 행동 진위 여부 판별이었음 원래는 우리 거가.
> - 에이전트가 파일 오픈 시 lnk 파일이나 프리패치는 상관없을 듯? 어떤 파일이 오픈을 한 흔적이 사람이 한 거냐 에이전트가 한 거냐 이거 판단 가능. CLI 아티팩트들 중에서 접근한 파일 목록 확인하는 기능. 시간대 → AI가 작업하면서 접근했을 수도 있다.
> - 시각화 → 파일 리스트 추가하기(사용자가 업로드한 거나 에이전트가 읽었거나 이런 접근한 파일들에 대한 리스트를 보여주기).
> - MCP로 접근한 프로그램들(프리패치에 남을 수 있음).
> - 나는 잘 사용 안 한 프로그램인데 → 에이전트가 사용 → 사람이 많이 사용한 것처럼 뜰 수 있음.
> - 혹시나 claude가 어떤 프로젝트 경로에 temp 경로 만들어서 뭘 만들고 지우고 하는데, 그 파일들을 복구(FAT32), 메타데이터도 중요.
> - 시각화 중요: 타임라인을 쭉 그려주는데 파이 그래프로 어떤 키워드를 많이 언급하고 있는지, 수사 관점에서 분석이 필요한 키워드가 있는지, 있다면 관련 대화를 특정 시간에 집중적으로 했는지, 아니면 오래오래 계속 얘기하고 있는지.
> - 시각화에서 파일 리스트도 올려주는 것도 좋을 듯(사용자가 업로드를 했든, LLM이 실제 읽든 접근한 파일들).
> - MCP로 접근한 건 프리패치로 남을 수 있음. 에이전트가 많이 사용해서 많이 사용한 것처럼 보여질 수 있음(실 사용자는 사용 안 하는데).
