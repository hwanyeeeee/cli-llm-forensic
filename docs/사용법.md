# clfx 사용법 (데모·시연용)

`clfx` = Claude Code 기록(`~/.claude`) 포렌식 CLI. **파싱 → 분석 → 질의 → 대시보드** 4단계.

> 설계 = `설계.md`, 출력 형식 = `event-schema.md`, 차별점 = `논문대비-신규발견.md`.

## 설치

```bash
pip install -e ".[dev]"
```

콘솔 스크립트가 PATH에 없으면 `python3 -m clfx.cli` 로 대체(아래 예시는 `clfx` 표기).

## 파이프라인

```
~/.claude  ──parse──▶  events.jsonl  ──analyze──▶  analyzed.jsonl  ──query──▶  답
                                                          └────────serve────▶  웹 대시보드
```

### 1. parse — 기록을 Event 스트림으로

```bash
clfx parse ~/.claude -o events.jsonl
```

- `~/.claude`(history.jsonl + projects/*.jsonl)를 읽어 한 줄=한 Event(JSONL)로.
- 각 Event는 `source{file,line}` 로 원본 위치를 보존(증거 역추적).
- paste 3사슬(`content`/`contentHash`→`paste-cache`/이미지 base64), §B 레코드(read/bash/write/response/prompt) 처리.
- `ts`는 ISO8601 UTC로 정규화(history epoch-ms도 통일).

### 2. analyze — 태깅·마스킹·귀속

```bash
clfx analyze events.jsonl -o analyzed.jsonl --root ~/.claude
```

- 시크릿 8종 + PII 탐지 → `secret`/`pii` 태그 + preview `‹secret›` 마스킹.
- `--root` 주면 `bypassPermissions` 세션의 read에 `bypass-mode` 태그(위험모드 귀속).
- `actor`(user/agent)는 파서가 확정한 것을 보존(여기서 안 바꿈).

> ⚠ `events.jsonl`/`analyzed.jsonl`은 마스킹 전 단계라면 시크릿 평문을 포함할 수 있다. 커밋 금지(`clfx-out/`는 gitignore됨).

### 3. query — 자연어 질의 (결정적 엔진)

```bash
clfx query analyzed.jsonl "누가 .env 읽었어?"
clfx query analyzed.jsonl "유출된 비밀 뭐야?"
clfx query analyzed.jsonl "2026-06-11 무슨 일 있었어 요약해줘"
clfx query analyzed.jsonl "타임라인 보여줘"
```

- 증거 주장은 **결정적 엔진**(검색·귀속·탐지) — LLM 없이도 동작·안 흔들림.
- 요약 요청("요약/정리")만 LLM 어댑터(없으면 digest fallback).
- 각 결과는 `[ts] actor/action target (file:line)` + 마스킹 preview.

### 4. serve — 웹 대시보드

```bash
clfx serve analyzed.jsonl --port 8787
# → http://127.0.0.1:8787  (Ctrl+C 종료)
```

- 로컬 read-only 뷰. 타임라인 **actor 색구분**(user=파랑 / agent=빨강 → A/B 사건 시각화).
- actor/action/tag 필터, 자연어 질의 박스(엔진 위임), secret/pii 하이라이트, 이벤트 클릭→`source(file:line)` 역추적.
- 검색·귀속·탐지는 전부 서버가 엔진 호출(JS 재구현 없음 = 증거 분기 방지). 의존성 0(stdlib만).

## A/B 데모 시연 흐름

1. `parse`→`analyze` 로 `analyzed.jsonl` 생성.
2. `serve` 로 대시보드 열기 → 타임라인에서 **A(user paste)**·**B(agent 자율 read)** 색으로 구분.
3. secret 뱃지 이벤트 클릭 → `source` 로 "이 유출이 이 파일 이 줄에서 왔다" 역추적.
4. 질의 박스: "누가 id_rsa 읽었어?" → actor=agent 규명(B), "유출된 비밀" → 노출 목록.
