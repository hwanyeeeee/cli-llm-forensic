## 0. 요약 — 무엇을 건드렸나 (범위)

`git status` 기준 **수정 파일은 3개뿐**, 전부 UI 작업영역(`clfx/web/static/`):

```
 M clfx/web/static/index.html
 M clfx/web/static/app.css
 M clfx/web/static/app.js
```

- **백엔드(엔진·API·서버·파서)는 한 줄도 수정 안 함.**
  `clfx/web/api.py`, `clfx/web/server.py`, `clfx/query/*`, `clfx/analyze/*`, `clfx/parser.py`, `clfx/event.py`, `clfx/cli.py` → 변경 없음(가이드 §2 역할분담 준수: UI팀원은 static 3파일만).
- 신규 파일 추가 없음(이 문서 제외). `clfx-out/`는 산출물이라 `.gitignore` 대상.
- **UI↔엔진 계약 유지**: 검색·귀속·탐지·집계 로직을 JS로 재구현하지 않음. 화면은 `/api/events`·`/api/query`를 호출·표시만.

즉 "도구의 기능/엔진 로직"이 바뀐 게 아니라, **화면(프론트) 표시층만 전면 개편**했다.

## 1. exe 도구인가? — 아니오 (아직)

| 항목 | 현재 상태 |
|---|---|
| 실행 형태 | 순수 Python 패키지 + stdlib 웹서버 |
| 실행 방법 | `python -m clfx.cli serve analyzed.jsonl --port 8787` → 브라우저 |
| exe 산출물 | **없음** — `packaging/`·`dist/`·`build/`·`*.spec` 전부 부재 |
| `packaging/launcher.py` | 아직 없음 |

- 클론에 들어온 `docs/exe-패키징-UI-가이드.md`는 exe를 **어떻게 만들지 설계만** 적힌 문서. PyInstaller `--onefile` + 내장 서버 + `webbrowser.open` 방식.
- exe 빌드/런처/`_MEIPASS` static 경로 보정은 **코어(엔진·패키징) 담당** 작업이며 **아직 미수행**.
- 우리 UI는 exe로 묶이면 그 안의 화면이 된다. 지금은 개발용으로 `clfx serve`로 띄워 확인(가이드 §7: UI 작업은 매번 exe 빌드 없이 serve로 검증).

## 3. 표시 정책상 바뀐 동작(엔진과 다른 점) — 주의

엔진 출력은 그대로 두고, **화면 표시 단계에서만** 다음을 적용했다. 엔진/저장 데이터는 변하지 않는다.

1. **시크릿(자격증명 패턴) 표시 제거**
   - `analyze`는 여전히 `secret` 태그를 산출한다(엔진 무수정). UI에서 `normalize()`가 표시용으로 `secret` 태그만 걸러낸다.
   - 이유: 회사마다 기밀 기준이 달라 도구가 "유출"을 단정할 수 없고, 정규식 패턴은 오탐 위험. "유출 판정"이 아니라 주체·권한 중심으로 표시.
   - 결과: 통계 타일·필터·배지에서 "시크릿" 제거, 대신 `bypass 모드`(권한 귀속) 노출.

2. **키워드(`kw`)는 UI측 파생 — placeholder**
   - 엔진 event에는 `kw` 필드가 없다. `deriveKw()`가 파일명·사전 매칭으로 임시 생성.
   - **계약상 임시 조치.** 코어가 `GET /api/keywords`(가이드 §3 예정)를 제공하면 그 집계로 교체해야 한다(집계는 엔진이 진실원천이어야 함).

3. **샘플 폴백(MOCK)**
   - 서버 미연결 시 내장 샘플로 화면을 채운다. 헤더 칩이 `샘플 데이터(서버 미연결)`로 표기되어 실데이터와 구분된다.

## 3.5 시크릿 탐지 기능 제외 — 상세 결정 근거

UI 개편 중 가장 중요한 정책 결정이라 별도로 기록한다.

### 배경
초기 UI에는 **"시크릿 노출"** 이 핵심 지표(통계 타일·필터칩·배지·히트맵 빨강 테두리)로 들어가 있었다. 이를 **UI 표시에서 전면 제외**하기로 결정.

### 제외 대상이 뭐였나
- 엔진 `clfx/analyze/secrets.py`의 **자격증명 정규식 8종**: `ssh_private_key`, `stripe(sk_live_)`, `aws_key_id(AKIA)`, `github_pat(ghp_)`, `openai_key(sk-)`, `npm_token(npm_)`, `aws_secret`, `db_password`(+`email`=PII).
- 매칭 시 `secret` 태그 부여 + preview `‹secret›` 마스킹.
- UI는 이를 "시크릿 노출 N건" 통계, 필터, 카드 배지, 히트맵 빨강 테로 강조했다.

### 왜 뺐나 (교수님 피드백 관점)
1. **교수님이 원한 "유출 확인" 메커니즘과 다르다.**
   - 피드백 §2: *"중요 파일이 사설 LLM에 올라갔을 수 있음 → **해시값으로 원본과 대조**해 유출 여부 확인."*
   - 이는 **특정 사내 기밀 파일의 동일성(콘텐츠 해시 매칭)** 기반 — "이 파일이 그대로 외부로 갔나"를 확정한다.
   - 반면 시크릿 탐지는 **범용 자격증명 정규식**이라 회사 기밀문서가 무엇인지 모른다. → "회사마다 기밀 기준이 다르고 도구가 판단할 수 없다"는 한계와 정확히 일치.
2. **형식 매칭이라 오탐·과장 위험.** 더미/예시 키도 패턴이 맞으면 잡힌다. "노출됨" ≠ "유출 사건". 단독 기능으로 내세우면 도구가 사건을 단정하는 과장이 된다(발표 §4 금지표와 충돌).
3. 도구의 역할은 **사실 집계(주체·권한·시각)** 까지이고, 유출·책임 판정은 사람 몫이라는 원칙과 더 맞다.

### 그래도 의미 있던 지점 (제거하되 인지)
- 주체 판별 보강(`secret + actor` = "에이전트가 자율로 .env 열람"), 트리아지 필터, 타임라인 앵커.
- 단, 이는 **보조 신호**일 뿐 "유출 판정"이 아니라서, 핵심 지표 자리에서 내렸다.

### 결정·구현
- **엔진은 무수정.** `analyze`는 여전히 `secret` 태그를 산출한다(증거 보존). UI `normalize()`가 **표시 단계에서만** `secret` 태그를 걸러낸다.
- 핵심 지표 자리는 **`bypass 모드`(권한 귀속)** 로 대체 — 행위 주체 판별에 직접 기여하는 지표.

### UI에서 실제로 바꾼 목록
| 위치 | 변경 |
|---|---|
| 통계 타일 | "시크릿 노출" → **"bypass 모드 읽기"** |
| 필터칩 | "시크릿" 칩 제거(A/B/bypass만) |
| 날짜 헤더 pill | "시크릿 N" → **"bypass N"**(보라) |
| 히트맵 | 시크릿일 빨강 테두리 제거 |
| 카드/상세 배지 | `secret` 배지 미표시(`normalize`에서 태그 제거) |
| AI 코파일럿 | 추천칩·응답·인사말의 "시크릿/유출" 문구 → bypass·중립으로 교체 |

### 향후 권장
- 진짜 "유출 확정" = **해시 대조**(사용자 업로드 원본 ↔ 사내 기밀 파일 해시 일치). 이게 교수님 피드백의 핵심이고 현재 빠져 있다.
- 코어가 해시 대조 기능을 추가하면 UI에 **"기밀 파일 해시 일치" 패널** 자리를 마련해 연결(시크릿 탐지의 대체가 아니라, 본래 의도한 기능).

## 4. 백엔드(코어)에 요청 예정인 신규 API

UI는 아래가 준비되면 fetch만 바꿔 연결한다(집계 로직은 엔진이 보유).

- `GET /api/keywords` → 키워드 빈도 집계(현재 UI 파생 대체)
- `GET /api/activity?by=day|month` → 활동량 집계(현재 UI가 events에서 직접 계산 중 → 엔진 위임 권장)
- 프롬프트 요약 → 기존 `/api/query` summarize 활용 또는 `/api/prompts/summary`

> 위 셋은 현재 UI가 `events`로부터 클라이언트 집계 중인 placeholder. 증거 분기 방지를 위해 최종적으로 엔진 집계로 옮기는 게 계약에 맞다.

## 5. 검증 결과 (fixture 8건 기준)

- `clfx serve` 기동 후 `/`·`/app.css`·`/app.js`·`/api/events`·`/api/query` 전부 **HTTP 200**.
- `/api/events` event 키(`action/actor/agent/preview/session/source/tags/target/ts`) ↔ `normalize()` 일치.
- `/api/query?q=타임라인 요약해줘` → `op=timeline, count=8, summary=true` 정상.
- 화면: 라이트 테마·달별 히트맵·날짜 아코디언·키워드 도넛/팝업·파일목록·코파일럿 정상 렌더, 실데이터 로드 확인.
