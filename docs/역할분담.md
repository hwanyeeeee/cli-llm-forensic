# 역할 분담

도구 개발은 **한 명(코어)이 전부** 한다 — 파싱·분석·질의 전 과정. 팀원 2명은 코드 말고 아래를 맡는다.

## 코어 — 도구 개발
파싱 → 분석 → AI 질의 전체. (`설계.md` 의 세 단계)

## 팀원 (miiin03, Chaeme22)

- **통제 데이터셋** — `스테이징-데이터셋-가이드.md` 대로, 정답을 아는 사건 데이터를 만든다.
- **검증** — 도구가 완성되면 그 데이터셋으로 정확도를 잰다 (시크릿 몇 개 맞췄나, 사건을 맞게 재구성하나).
- **발표 자료** — `발표-가이드라인.md` 대로 만든다.
- **다른 환경 교차확인** — `논문대비-신규발견.md` 의 표를 자기 PC에서 채운다.

## 합의할 것 하나

도구 출력과 데이터셋 정답이 **같은 형식**을 써야 비교가 된다. → `event-schema.md` 의 `actor`·`action`·`target` 을 양쪽이 그대로 쓴다. 이거 하나만 맞추면 나머지는 따로 진행.
