## 0. 분담 원칙

핵심 파이프라인(`수집→파싱→정규화→복구→상관`)은 **밀결합** → 쪼개면 조율비용 > 이득. **코어는 1인이 직렬로**(빠름·일관). 팀원은 **코어와 분리돼 안 막히는** 실험·검증·발표를 병렬로. 이게 "한쪽이 빠르게"와 "2인 기여(루브릭)" 둘 다 잡는 구성.

| | **개발자 (코어)** | **팀원 (실험·검증·발표)** |
|---|---|---|
| 강점 | Juhwan — SQLite 카빙·integrity 전이 | 팀원 — 실험설계·문서·발표 |
| 루브릭 | #2 구현·정확성, #5 기법 | **#3 검증(최고배점)**, #2 실데이터, #4 발표 |

## 1. 개발자 — 도구 전체 (코어 직렬)

spec §5~6 전부 한 손으로:
- **F1 수집** — WSL/네이티브 경로 자동탐색 + ext4.vhdx 읽기 + 해시
- **F2 파서** — Claude/Codex/Gemini transcript·config·credential (버전핀)
- **F3 정규화** — Event 스키마(`docs/event-schema.md`)
- **F4 시크릿/PII 탐지** · **F5 귀속**
- **F6 복구** — file-history·삭제 카빙(ext4)·ChromaDB
- **F7 상관·재구성** — 다층·WSL2 경계·MITRE·재구성등급
- **F8 뷰어/리포트** · **F9 AI 질의(보조, provenance 강제)** · **F10 무결성**
- **채점 하네스** — 도구 출력 ↔ ground-truth 매칭(event-schema §3) → precision/recall·재구성등급

> **도구 전체를 코어(1인)가 구현.** 팀원은 데이터셋·검증실행·발표(코드 X).

## 2. 팀원 — 실험·검증·발표 (코어 안 막는 병렬)

| 작업 | 산출 | 의존 |
|---|---|---|
| **① 통제 스테이징 데이터셋(Nx·내부자)** | evidence + ground-truth.json + hashes | 없음(독립) — `docs/스테이징-데이터셋-가이드.md` 그대로 |
| **② 평가 메트릭** | precision/recall·재구성등급·무결성검증 표 | 도구 *완성 후* 실측 |
| **③ 무료도구 대비** | OSS 세션뷰어 vs 우리(분석·복구·무결성) 표 | 일부 독립(OSS 조사) |
| **④ 발표자료 초안** | 슬라이드 — `docs/발표-가이드라인.md` 따라 | 독립(데모 영상만 도구 후) |
| **⑤ Q&A 시트·기여 매트릭스** | 문서 | 독립 |

**핵심:** ①③④⑤는 **지금 바로 시작 가능**(독립). ②만 도구 완성 후 실측 → 팀원은 *데이터셋·방법론·발표 골격 먼저*, *실측은 마지막*. 절대 안 막힘.

---

## 3. 유일한 합의점 = Event 스키마 → **`docs/event-schema.md` (canonical, 확정 v1.0)**

팀원의 ①(ground-truth.json `attack_events`)·F4·채점 하네스가 이 스키마와 *글자 그대로 같은 enum*을 써야 대조됨. **`docs/event-schema.md`가 유일 원천** — 여기 인라인 복제 금지(드리프트 방지).

핵심(상세는 그 파일):
- enum 고정: `actor`(user/agent/subagent/hook/system)·`channel`·`effect`·`evidence_class`·`resource_type`.
- **매칭 계약**(§3): resource 정규화(file=realpath/home상대, cmd=argv0+대상) + (effect,resource_type,resource)+window 1:1.
- ground-truth `attack_events`도 같은 enum(§4 예시). `system` 이벤트는 `evidence_class:host_confirmed`(호스트 아티팩트로 확증).
- 변경은 그 파일 PR로만.

## 4. 순서

1. **(공동·30분)** Event 스키마 합의 + 미해결질문 5개(ext4.vhdx·Codex암호화·범위·제품명·역할) 확정.
2. **(팀원 즉시·병렬)** 스테이징 데이터셋 ① + OSS 조사 ③ + 발표 골격 ④.
3. **(개발자)** ext4.vhdx PoC → Claude 파서 → 정규화 → 복구 → 상관 → AI → 뷰어. (직렬)
4. **(개발자→팀원)** 도구 동작본 전달.
5. **(팀원)** 데이터셋으로 ② 실측(메트릭·무결성) + 데모영상 + 발표 완성.
6. **(공동)** 리허설·Q&A·기여매트릭스.

**리스크:** 팀원 ②는 도구에 의존 → 도구가 늦으면 메트릭 밀림. 완화 = 팀원이 ①③④⑤를 먼저 다 끝내두고 대기, 도구 나오면 ②만 빠르게.
