# 역할 분담

도구 개발은 **한 명(코어)이 전부** 한다 — 파싱·분석·질의 전 과정. 팀원 2명은 코드 말고 아래를 맡는다.

## 코어 — 도구 개발
파싱 → 분석 → AI 질의 전체. (`설계.md` 의 세 단계)

## 팀원 (miiin03, Chaeme22)

- **다른 환경 교차확인** — `논문대비-신규발견.md` 의 §A·§B·§C "검증 명령"을 자기 PC에서 돌려 "내 결과" 칸을 채운다(n=2 일반화). **도구 필요 없음 — 지금 바로 가능.**
- **발표 자료** — `발표-가이드라인.md` 대로 초안.
- **통제 데이터셋** — `스테이징-데이터셋-가이드.md` 대로, 정답을 아는 A/B 사건 데이터를 만든다.
- **검증** — 도구가 완성되면 그 데이터셋으로 정확도를 잰다 (시크릿 몇 개 맞췄나, A/B 주체를 맞게 귀속하나).

### 작업 순서 (타이밍)
| 시점 | 팀원이 할 일 |
|---|---|
| **지금** (코어가 도구 개발 중) | ① 신규발견 교차확인(내 결과 채우기) + ② 발표 초안 — 둘 다 도구 없이 가능 |
| **나중** (도구 완성 후) | ③ 통제 데이터셋 + ④ 도구로 채점(precision/recall·주체 귀속 정확도) |

## 합의할 것 하나

도구 출력과 데이터셋 정답이 **같은 형식**을 써야 비교가 된다. → `event-schema.md` 의 `actor`·`action`·`target` 을 양쪽이 그대로 쓴다. 이거 하나만 맞추면 나머지는 따로 진행.
