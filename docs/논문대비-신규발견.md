## 진짜 차별점 (실측으로 확인됨)

### A. 붙여넣은 본문이 프롬프트와 **분리 저장**된다 — `pastedContents`

논문은 `history.jsonl` 을 "프롬프트 기록"으로 본다. 그런데 실제 구조는 다르다: **프롬프트 표시 칸(`display`)엔 붙여넣은 본문이 없고**, 쪽지(placeholder)만 있다. 진짜 본문을 찾으려면 최대 3단계를 따라가야 한다.

```
display                = "[Pasted text #1 +4 lines] 이러헥 떠"            ← 쪽지(placeholder)만
pastedContents['1']    = {id, type:"text", content 또는 contentHash}     ← 같은 줄, 본문 또는 해시
paste-cache/<hash>.txt = 실제 본문                                       ← 해시만 있으면 여기
```

- **본문이 항상 `content` 에 있는 건 아니다**(실측): `pastedContents` 항목 2239개 중 `content` 직접 보유 892개 / `contentHash` 만 1347개 → 후자는 본문이 `~/.claude/paste-cache/<hash>.txt` 에 따로 있음(369개 캐시 파일 확인).
- **`pastedContents` 는 텍스트 전용**(2239개 전부 `type:text`). 붙여넣은 **이미지**는 history.jsonl 안 거치고 **projects transcript 레코드 안에 base64로 inline** 저장된다: `message.content[]` 의 `type:image` 파트 → `source.data`(base64 PNG), `imagePasteIds` 가 `[Image #N]` 쪽지와 매핑. base64 디코드하면 원본 PNG 복원(실측: 1100×1318 등 정상 추출).
- 포렌식 의미: 프롬프트(`display`)만 읽으면 붙여넣은 내용(=유출 후보)을 통째로 놓친다. `display → pastedContents → paste-cache` 사슬을 끝까지 따라가야 본문이 보인다.
- 실측: `history.jsonl` 10801줄 중 **2153줄**이 `pastedContents` 보유.
- 검증 명령:
  ```bash
  python3 -c "import json,os; rows=[json.loads(l) for l in open(os.path.expanduser('~/.claude/history.jsonl'),encoding='utf-8',errors='ignore') if l.strip()]; pc=[r for r in rows if r.get('pastedContents')]; items=[v for r in pc for v in r['pastedContents'].values()]; print('총',len(rows),'/ pasted줄',len(pc),'/ content직접',sum(1 for v in items if v.get('content')),'/ hash만',sum(1 for v in items if not v.get('content') and v.get('contentHash')))"
  ```

### B. 논문 이후 생긴 **새 증거 레코드 타입** — `projects/*.jsonl`

논문 버전(2.1.72)엔 없던 레코드 타입들이 transcript에 추가됐다. 포렌식 가치 큰 것:

| 레코드 타입                               | 무엇                                        | 왜 증거               |
| ------------------------------------ | ----------------------------------------- | ------------------ |
| `toolUseResult`                      | 에이전트가 읽은 파일의 실제 내용                        | 무슨 비밀/소스를 봤는지      |
| `file-history-snapshot` (7,179)      | 편집 전 파일 백업                                | 원본 상태 복원           |
| `permission-mode` (11,990)           | 위험모드(`--dangerously-skip-permissions`) 여부 | 공격 가능성의 핵심 신호      |
| `agent-name` (5,343) / `isSidechain` | 어느 서브에이전트가 한 행동인지                         | 행위 귀속(attribution) |
| `thinking` (37,659)                  | 에이전트 내부 추론                                | 의도·계획              |

- 검증 명령:
  ```bash
  grep -rhoE '"type":"(toolUseResult|file-history-snapshot|permission-mode|agent-name|thinking)"' ~/.claude/projects 2>/dev/null | sort | uniq -c
  ```

## 검증했지만 차별점이 아닌 것 (정직 — 같은 착오 반복 방지)

이 항목들은 처음엔 "신규 발견"으로 적었으나 **실데이터로 확인하니 틀렸거나 약했다.** 발표에 차별점으로 넣지 말 것.

| 항목 | 처음 생각 | 실측 결과 | 판정 |
|---|---|---|---|
| 버전 드리프트 (2.1.72→2.1.178) | 신규 발견 | 시간 지나 버전 오른 것뿐 | **배경**일 뿐, 기여 아님 |
| Codex가 Fernet 안 씀(평문화) | 평문으로 읽힘 | 222개 세션 중 **193개에 `encrypted_content:"gAAAAA"`(Fernet) 여전히 존재**. 추론은 아직 암호화, 겉(프롬프트·cwd·모델)만 평문 | **거짓** — 삭제 |
| Windows+WSL2 이중 저장소 | 플랫폼 차별점 | 윈도우+WSL 둘 다 설치 시 생기는 **설치 선택**, 파일 형식 동일 | **차별점 아님** |
| `agents/`·`teams/` = 멀티에이전트 이미 존재 | 신규 아티팩트 | 두 폴더 **모두 비어 있음(empty)**. 진짜 서브에이전트 흔적은 폴더가 아니라 §B의 `agent-name`/`isSidechain` 에 있음 | **거짓** — §B에 흡수 |

> 교훈: 디렉토리·파일이 "존재"하는 것과 "증거를 담는" 것은 다르다. 빈 폴더는 증거가 아니다. 항상 **안의 내용**을 확인하고 적는다.
