# 논문 대비 신규 발견 + 팀원 머신 교차검증

날짜: 2026-06-16
근거 논문: Kim & Jeong, "From assistant to accomplice" (SSRN abstract_id=6725750, Ubuntu·v2.1.72 등) — `docs/Ref/`
관측 머신: 개발자 PC (Windows 11 + WSL2). **버전·아티팩트는 머신마다 다를 수 있음 → 팀원 머신서도 돌려 n=2로 굳히자**(논문 "n=1 일반화" 비판 방어).

> 사용법: 팀원은 *자기 머신*(WSL2)에서 아래 "검증 명령"을 돌려 "내 결과" 칸을 채운다. 같으면 발견 일반화, 다르면 *그 차이도 데이터*(버전핀).

### §검증3 (projects 레코드타입)
```bash
f=$(find ~/.claude/projects -name "*.jsonl" | head -1)
python3 -c "
import json;from collections import Counter
c=Counter()
for l in open('$f',encoding='utf-8',errors='ignore'):
    try:c[json.loads(l).get('type')]+=1
    except:pass
print(dict(c))"
# 기대: last-prompt, file-history-snapshot, agent-name, permission-mode, user, assistant, system 등
```

## 이 표가 왜 중요 (발표·논문)
- **차별 기여의 증거** — "논문 이후 에이전트가 진화해 *새 아티팩트 클래스*(체크포인트·붙여넣기캐시·서브에이전트)가 생겼고, 우리가 이를 최초로 포렌식 매핑" → spec §2 차별점의 *실측 근거*.
- **n=2 검증** — 개발자+팀원 두 머신서 일치 확인 → "n=1 일반화" 비판 방어. 차이가 나면 그 자체가 *버전·플랫폼 의존성* 데이터(논문도 이걸 한계로 인정).
- **파서 설계 입력** — 코어 F2 파서가 이 신규 타입/경로를 다 다뤄야 함.

## 주의 (정직)
- 이 아티팩트는 **버전·벤더 의존**(공식 API 계약 없음, 리버스). 버전핀 명시.
- 팀원 결과가 다르면 *틀린 게 아니라* 버전 차이 — 버전과 함께 기록.
- 자기 머신 `~/.claude` 점검 시 **본인 실데이터** → 값 출력 말고 *키/구조/존재여부*만(시크릿 노출 주의).
