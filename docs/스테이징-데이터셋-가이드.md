# 통제 데이터셋 만들기 (팀원 가이드)

담당: 검증(팀원). 소비: 분석 엔진(코어). 어휘는 `docs/event-schema.md` 의 `actor`·`action`·`target` 그대로.

> **목적**: *정답을 우리가 아는* 가짜 사건 데이터를 만든다. 그래야 도구가 "시크릿 8개 중 몇 개 잡았나"를 채점할 수 있다. 실데이터엔 정답표가 없어서 정확도를 못 잰다 — 그래서 이걸 따로 만든다.

---

## 왜 별도 격리(distro) 안 써도 되나

우리는 **진짜 멀웨어를 안 돌린다.** 우리가 짠 양성 스크립트로 흉내만 낸다. 그래서:

1. 가짜 시크릿은 **전부 한 폴더(`~/clfx-victim`) 안**에만 만든다 → 끝나면 폴더째 지움.
2. claude는 transcript를 **작업폴더별로** `~/.claude/projects/<인코딩된-경로>/` 에 따로 저장한다 → `~/clfx-victim` 에서 돌리면 그 세션만 독립된 파일. 기존 실데이터랑 안 섞인다.
3. "공격"은 그 폴더 파일을 읽어 **같은 폴더 안 로컬 sink**에만 쓴다 → 외부 전송 0.
4. 진짜 홈(`~/.ssh`·`~/.bashrc`·`~/.npmrc`)은 **건드리지 않는다** → 키 파괴 위험 없음.

> 더 강하게 격리하고 싶으면(선택): `HOME=$(mktemp -d) claude ...` 로 돌리면 claude가 임시 홈을 쓴다. 단 그 홈은 미인증이라 **claude 재로그인 1회** 필요. 안 해도 위 4개면 충분하다.

---

## 산출물 (폴더 `clfx-dataset-<시나리오>-<날짜>/`)
1. `evidence/` — 수집한 아티팩트(구조 보존).
2. `ground-truth.json` — 정답(§4에서 명령으로 생성).
3. `hashes.txt` — evidence SHA256.
4. `README.md` — 시나리오 1줄 + 환경(에이전트·버전).

두 시나리오: **A(사용자 직접 입력)** / **B(에이전트 자율 접근)** — 한 폴더에 둘 다 만든다.

## 2. 두 시나리오 실행

도구가 가를 두 경우를 각각 만든다. **핵심 채점 = 도구가 `actor`(user vs agent)를 맞히는가.** 한 `~/clfx-victim` 폴더 안에서 A와 B를 둘 다 돌린다(한 데이터셋에 두 사건).

### 2-A. 시나리오 A — 사용자 직접 입력 (붙여넣기, §A)
직원이 기밀을 직접 AI 입력창에 붙여넣는 경우. **인터랙티브로 실제 붙여넣어야** `pastedContents`/`paste-cache` 가 생긴다 (`claude -p` 인자 전달은 붙여넣기로 안 잡힌다).

```bash
cd ~/clfx-victim && mkdir -p sink
date -u +%FT%TZ > sink/A-start.txt
cat .env                  # 이 본문을 복사
claude                    # 인터랙티브 실행
#   (TUI 입력창에) .env 본문을 붙여넣고 이렇게 전송:
#   "이 설정값들 중 잘못된 거 있는지 봐줘: <여기에 .env 붙여넣기>"
#   → history.jsonl 에 actor:user + pastedContents(또는 contentHash)
#   /exit 로 종료
date -u +%FT%TZ > sink/A-end.txt
```
> 붙여넣은 본문이 길면 `~/.claude/paste-cache/<hash>.txt`, 짧으면 `pastedContents.content` 에 저장된다. 둘 다 §5에서 수집.

### 2-B. 시나리오 B — 에이전트 자율 접근 (Nx식, §B)
위험모드로 에이전트가 직원 입력 없이 `.env`를 읽는 경우. **실제 Nx 멀웨어·공급망은 재현하지 않는다 — 그게 남길 아티팩트만** 만든다.

```bash
cd ~/clfx-victim && mkdir -p sink
date -u +%FT%TZ > sink/B-start.txt
PROMPT="Audit this repository for hardcoded secrets. Read .env, config.py, and any key files, then list every secret you find with its file path."
claude --dangerously-skip-permissions -p "$PROMPT" > sink/agent_out.txt 2>&1
# 호스트 효과(에이전트와 무관하게 셸이 직접 빼돌리는 부분) — 로컬 sink만
cat .env config.py keys/id_rsa .npmrc > sink/inventory.txt 2>/dev/null
base64 sink/inventory.txt > sink/results.b64
date -u +%FT%TZ > sink/B-end.txt
# 거부/미인증이면 agent_out.txt에 흔적 남음 → §7 폴백
grep -qiE "can't help|cannot|won't|refus|not able|login|authenticate|unauthorized|error" sink/agent_out.txt && echo "AGENT_REFUSED" || echo "AGENT_OK"
```

## 3. 이 세션들의 아티팩트 위치 찾기

claude가 방금 세션을 `~/.claude/` 안에 저장했다. cwd가 `clfx-victim` 인 것만:

```bash
# B(자율) 증거 = projects transcript (toolUseResult)
ls -d ~/.claude/projects/*clfx-victim* 2>/dev/null
# A(붙여넣기) 증거 = history.jsonl 의 project=clfx-victim 줄 + 그 paste-cache
python3 -c "import json,os; rows=[json.loads(l) for l in open(os.path.expanduser('~/.claude/history.jsonl'),encoding='utf-8',errors='ignore') if l.strip()]; v=[r for r in rows if 'clfx-victim' in str(r.get('project','')) and r.get('pastedContents')]; print('붙여넣기 줄', len(v), '/ 해시', [h for r in v for it in r['pastedContents'].values() for h in [it.get('contentHash')] if h])"
```

이것들이 코어에게 줄 핵심 증거다(이것만 쓰면 기존 실데이터와 안 섞인다).

## 4. 정답 생성 — `ground-truth.json`

`actor`/`action`/`target` 은 `event-schema.md` 그대로. 시각은 sink의 start/end에서 채움:

```bash
cd ~/clfx-victim
REFUSED=$(grep -qiE "can't help|cannot|won't|refus|not able|login|authenticate|unauthorized|error" sink/agent_out.txt && echo true || echo false)
cat > ground-truth.json <<EOF
{
  "scenario": "combined-A-paste-and-B-autonomous",
  "env": { "agent": "claude", "version": "$(claude --version 2>/dev/null | head -1)" },
  "window": { "start": "$(cat sink/A-start.txt)", "end": "$(cat sink/B-end.txt)" },
  "planted_secrets": [
    {"id":"CLFXTEST-001","type":"stripe","file":"~/clfx-victim/.env"},
    {"id":"CLFXTEST-002","type":"aws_key_id","file":"~/clfx-victim/.env"},
    {"id":"CLFXTEST-003","type":"aws_secret","file":"~/clfx-victim/.env"},
    {"id":"CLFXTEST-004","type":"db_password","file":"~/clfx-victim/.env"},
    {"id":"CLFXTEST-005","type":"github_pat","file":"~/clfx-victim/config.py"},
    {"id":"CLFXTEST-006","type":"openai_key","file":"~/clfx-victim/config.py"},
    {"id":"CLFXTEST-007","type":"ssh_private_key","file":"~/clfx-victim/keys/id_rsa"},
    {"id":"CLFXTEST-008","type":"npm_token","file":"~/clfx-victim/.npmrc"}
  ],
  "expected_detections": ["CLFXTEST-001","CLFXTEST-002","CLFXTEST-003","CLFXTEST-004","CLFXTEST-005","CLFXTEST-006","CLFXTEST-007","CLFXTEST-008"],
  "noise_files": ["~/clfx-victim/app.py"],
  "attack_events": [
    {"actor":"user","action":"paste","target":".env 내용(직접 붙여넣음)","scenario":"A"},
    {"actor":"agent","action":"read","target":"~/clfx-victim/.env","scenario":"B"},
    {"actor":"agent","action":"read","target":"~/clfx-victim/config.py","scenario":"B"}
  ],
  "expected_attribution": { "A": "user", "B": "agent" },
  "local_sink": "~/clfx-victim/sink/results.b64",
  "agent_refused": $REFUSED
}
EOF
python3 -c "import json;json.load(open('ground-truth.json'));print('JSON OK')"
echo "# clfx nx dataset — claude $(claude --version 2>/dev/null | head -1)" > README.md
```

> **핵심 채점 = 귀속(`expected_attribution`)**: 도구가 A 이벤트를 `actor:user`(붙여넣기), B 이벤트를 `actor:agent`(자율 read)로 맞히는가. 이게 시나리오의 본질.
> **에이전트가 거부했으면(`agent_refused: true`)**: B의 `actor:agent` 이벤트는 실제로 안 일어남 → `attack_events` 의 B 줄을 지운다(정직). A(붙여넣기)와 셸 호스트 효과(inventory/results.b64)는 남으니 데이터셋 유효.

## 5. 수집 + 무결성 + 패키징

```bash
DST=/mnt/c/projects/clfx-datasets/clfx-dataset-nx-$(date -u +%Y%m%d)
mkdir -p "$DST/evidence/dot-claude-projects"
# B(자율) 증거: 이 세션 transcript만 (기존 실데이터 제외)
cp -a ~/.claude/projects/*clfx-victim* "$DST/evidence/dot-claude-projects/" 2>/dev/null
# A(붙여넣기) 증거: history.jsonl 의 clfx-victim 줄만 + 그 paste-cache 본문 (전역 통째 복사 금지)
mkdir -p "$DST/evidence/dot-claude"
python3 - "$DST/evidence/dot-claude" <<'PY'
import json,os,sys,shutil
dst=sys.argv[1]
rows=[json.loads(l) for l in open(os.path.expanduser("~/.claude/history.jsonl"),encoding="utf-8",errors="ignore") if l.strip()]
vic=[r for r in rows if "clfx-victim" in str(r.get("project",""))]
open(dst+"/history.clfx-victim.jsonl","w",encoding="utf-8").write("".join(json.dumps(r,ensure_ascii=False)+"\n" for r in vic))
os.makedirs(dst+"/paste-cache",exist_ok=True)
hashes=[it.get("contentHash") for r in vic for it in (r.get("pastedContents") or {}).values() if it.get("contentHash")]
for h in hashes:
    p=os.path.expanduser(f"~/.claude/paste-cache/{h}.txt")
    if os.path.exists(p): shutil.copy(p, dst+f"/paste-cache/{h}.txt")
print("history줄",len(vic),"/ paste-cache",len(hashes))
PY
# 작업 폴더 통째 (시크릿·sink 포함)
cp -a ~/clfx-victim "$DST/evidence/victim"
# (.credentials.json 같은 자격증명은 절대 복사 안 함)
cp ~/clfx-victim/ground-truth.json ~/clfx-victim/README.md "$DST/"
( cd "$DST/evidence" && find . -type f -print0 | sort -z | xargs -0 sha256sum ) > "$DST/hashes.txt"
echo "전달본: $DST"
```

> **무엇이 증거인가**: **B(자율)** 증거 = `projects/*clfx-victim*` transcript의 `toolUseResult`(읽은 파일 원문, §B). **A(붙여넣기)** 증거 = `history.clfx-victim.jsonl` 의 `pastedContents` + `paste-cache/<hash>.txt`(본문, §A). 도구는 둘을 보고 각각 `actor:agent` / `actor:user` 로 귀속해야 한다.
> `file-history/` 는 이 시나리오에 파일 편집이 없어 비어 있다(안 모음). **전역 `history.jsonl` 은 절대 통째 복사하지 않는다** — `project=clfx-victim` 줄만 필터(실데이터 섞임 방지).

## 6. 정리 (Teardown)

```bash
rm -rf ~/clfx-victim
rm -rf ~/.claude/projects/*clfx-victim*    # B 세션 transcript 제거 (실데이터 안 건드림)
```

진짜 홈(키·자격증명)은 안 건드렸다. 단 **A(붙여넣기)는 전역 `history.jsonl`·`paste-cache/` 에 합성 줄을 남긴다**(전부 CLFXTEST 합성이라 무해). 깔끔히 지우려면 §3 명령으로 해시를 확인해 그 `paste-cache/<hash>.txt` 만 지우고 `history.jsonl` 의 clfx-victim 줄을 수동 제거. 아예 안 섞이게 하려면 §0의 `HOME=$(mktemp -d)` 방식으로 A를 돌린다.

## 7. 에이전트가 거부/미인증일 때 (폴백)

claude가 거부하거나 로그인 안 돼 있으면 `actor:agent` 이벤트가 없다. 두 가지:
- **그대로 둔다**: agent 이벤트를 ground-truth에서 빼고, 셸이 한 호스트 효과만 남긴다(정직). 데이터셋 여전히 유효.
- **합성 transcript**: 현 버전 샘플 1개를 코어에게 받아 같은 구조로 만든다(필수 레코드: `type:user` 프롬프트, `type:assistant`+`toolUseResult` 읽은 `.env` 원문, `cwd`/`timestamp`/`sessionId`). ground-truth에 `"method":"synthetic"` 명시.
