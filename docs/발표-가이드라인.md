## 0. 관통하는 한 줄

> **의심 → 위협 실재(예방은 뚫린다) → 사후 조사 도구 없음 → 우리 도구(사후 포렌식) → 입증(데모) → 현장 활용 → 기여**

- 핵심 질문 하나로 발표 전체를 꿴다: **"기밀이 외부 AI로 나갔다. 직원이 직접 넣었나(A), AI가 자율로 했나(B)?"**
- 데모(입증)가 클라이맥스. 도구가 **파싱→분석→질의** 3산출물로 A/B를 가르는 장면이 핵심.
- 도구는 *누가 했나(주체)*를 증거로 밝힌다. 책임·징계 *판단*은 사람 몫 — 이 선을 지키는 게 신뢰.

## 1. 통합 시나리오 (발표 전체의 뼈대)

**상황**: 한 직원이 기밀 유출로 의심받는다. 회사 정보보안팀 또는 외부 디지털포렌식 용역업체가 그의 에이전트 기록을 조사한다.

| | A. 사용자 직접 입력 | B. 에이전트 자율 접근 |
|---|---|---|
| 행위 | 직원이 기밀을 **직접 붙여넣음** | 위험모드·악성 패키지(Nx식)가 **직원 입력 없이** 에이전트로 `.env` 읽음 |
| 증거 | `history.jsonl`/`paste-cache` 붙여넣기 본문 | `projects/*.jsonl` `toolUseResult`+`permission-mode` |
| schema | `actor:user`, `action:paste` (§A) | `actor:agent`, `action:read` (§B) |
| 외부 전송 | ⭕ 제3자 AI 서버로 (규정 위반) | ⭕ 제3자 AI 서버로 (규정 위반) |
| 책임 | 사람이 **직접** 위반한 증거 | 사람이 **직접 한 게 아님** (과실·악성패키지 피해는 별개) |

**외부 AI 서버(Anthropic/OpenAI/Google)로 나간 건 양쪽 다 동일 — 규정 위반은 둘 다 성립(경쟁사 무관).** 도구의 역할 = "유출 여부"가 아니라 **`actor`(누가 그 행위를 했나)를 증거로 밝히는 것.** 책임·징계 판단은 도구 밖(인사·법무·수사).

## 3. 슬라이드별 작성 지침

**#1 훅** — 통계·정의로 열지 마라. *상황*으로. "어느 날, 회사 기밀이 외부 AI 서버에서 발견됐다. 용의자는 한 개발자. 그런데 — 그가 직접 넣었을까, 그의 AI가 멋대로 했을까?" 한 문장 + 한 이미지.

**#3 위협·예방한계** — 각 사건 *출처 명시*. 두 등급으로 나눠 말하되 *실제 입증된 것*을 근거로:
- **실제 사건/취약점**: 삼성 직원 ChatGPT 소스코드 입력(2023, A형) · Nx 2,180계정 공급망 무기화(2025, B형) · Claudy Day(Oasis Security, 2026, 프롬프트 주입→`api.anthropic.com` 유출, 패치됨) · Rehberger code-interpreter 유출(2025, HackerOne).
- 메커니즘 분석(DNS/인코딩 우회 등)은 "어떻게 가능한지" 보조 설명으로만 — 실제 사건과 섞지 말 것.
- 한 줄: **"예방형 DLP가 있어도 허용된 API 엔드포인트로 빠져나가면 안 보인다. 그래서 사고가 실제로 났다."**

**#4 갭** — 가장 중요한 설득. 핵심 한 줄: **"예방은 불완전해 사고가 난다. 그런데 사고 *후* 그 에이전트 기록을 파싱·분석하는 도구가 없다 — 지금은 수작업 grep, 프록시 로그는 내용도 안 보인다. 논문조차 `projects/` 내부를 안 깠다."** (우리는 예방이 아니라 *사후 포렌식* 층.)

**#5 차별** — 표 한 장. 행 = 기능(자동화 · §A 붙여넣기 분리저장 파싱 · §B 증거 레코드 타입 · 행위 주체 귀속 · 무결성 해시), 열 = 우리 / Kim&Jeong / OSS 뷰어. 우리만 ✅ 채워진 칸이 차별점.

**#7 데모(클라이맥스)** — 라이브 위험 → **녹화 영상 + 라이브 시도** 병행. **두 사건을 나란히**, 각각 우리 도구의 **3산출물**을 차례로 보여준다:
1. **파싱한 것** — 원본 jsonl이 구조화된 이벤트로 (시각·actor·action·target).
2. **분석한 것** — 타임라인 + 시크릿 탐지 + 주체 귀속(actor).
3. **질의 가능한 것** — "누가 .env를 외부로 보냈어?" 자연어 질문 → 답 + 원본 줄 추적.

증거 등급으로 내레이션(과장 금지):
- **A(직접 입력):** `history.jsonl` `actor:user`·`action:paste` — 직원이 직접 붙여넣음(본문 `paste-cache/<hash>.txt`, 출처 표기).
- **B(에이전트 자율):** `projects/<uuid>.jsonl:42` `actor:agent`·`toolUseResult`(.env 원문)+`permission-mode:bypassPermissions` — 직원 입력 없이 에이전트가 읽음.
- **공통:** 둘 다 외부 AI로 전송됨(유출=사실). 도구가 가르는 건 *주체*지 유출 여부가 아님.
→ "도구는 *누가 했나*를 증거로 보여준다. 책임 *판정*은 사람이 한다" — 이 선이 곧 엄밀성.

**#9 실제 현장 활용** — *누가 쓰나*를 구체적으로. "사내 정보보안팀 또는 외부 디지털포렌식 용역업체가, 유출 의심 직원의 단말을 받아 에이전트 기록을 우리 도구로 분석한다. 출력 = 무엇이(시크릿 인벤토리)·언제·**누가(A 직접 입력 vs B 에이전트 자율)**, 전부 원본 줄 추적." 이 증거 패키지로 **책임 소재의 출발점**을 잡는다. 징계·처벌은 인사·법무가 결정(도구는 근거, 결정은 사람). 부수 효과: 공급망 공격(B)이면 직원 누명도 벗긴다.

## 4. 발표자 내부 체크 — 절대 금지 표현 (슬라이드 아님, 즉시 깨지는 과장)

주체·책임을 다루는 도구라 *단정*과 *과장*이 특히 위험하다. 아래는 입에 담지 말 것.

| ❌ 금지 | ✅ 대체 |
|---|---|
| "CLI 에이전트는 *못 막는다 / 완전 사각*" | "예방은 *불완전*(허용 엔드포인트·우회로 뚫림). 그리고 *사후 조사* 도구가 없다" |
| "도구가 직원의 *고의·유출 목적을 증명*한다" | "도구는 `actor`(user/agent) *주체*를 증거로 밝힘. 고의·책임 판정은 조사관" |
| "B면 직원 *무죄*다 / 유출 없었다" | "B도 유출은 발생(규정 위반). *직접 한 게 아닐* 뿐 — 위험모드·악성패키지 과실은 별개" |
| "경쟁사로 유출됐다" | "제3자 AI 제공자로 *전송이 기록됨* — 경쟁사 도달은 별개" |
| "AI가 사건을 분석·판단" | "결정론적 파싱이 증거, AI는 *보조 해석*(provenance 추적)" |
| "실시간 탐지/방어/막는다" | "*사후* 포렌식 재구성(예방 아님, EDR/DLP 아님)" |
| "실제 Nx 사건을 분석" | "Nx식 공격이 *남길 아티팩트*를 통제 재현해 분석(멀웨어 자체는 재현 안 함)" |

## 6. Q&A 예상 (시트로 준비)

1. "DLP·보안 솔루션 있는데 왜 필요?" → 그건 *예방*(불완전, CLI에 구멍). 우리는 *사후 포렌식* — 사고 후 무엇이·누가·언제를 재구성. 층이 다름.
2. "OSS 세션 뷰어랑 뭐가 달라?" → 단순 열람 vs *사건 재구성·주체 귀속* (논문 미열거 아티팩트 파싱·시크릿 탐지·무결성·AI 질의).
3. "AI 신뢰성은? AI로 AI 잡는 거 아니냐?" → AI는 보조, 모든 결론 원본 추적(provenance), 결정론적 파싱이 1차.
4. "B(에이전트 자율)면 직원은 무죄냐?" → 아니다. **유출은 발생(규정 위반).** 직원이 *직접 한 게* 아니란 정황일 뿐 — 위험모드·악성패키지 과실은 별도. 도구는 주체 증거를 제시, 판단은 조사관/법무.
5. "직원이 '에이전트가 했다'고 발뺌하면?" → `actor:user`·`action:paste` 가 남으면 직접 입력의 증거. 그게 A/B를 가르는 핵심.
6. "오탐 많지 않냐(시크릿 탐지)?" → precision/recall 측정값 제시 + 컨텍스트 기반.
7. "버전 바뀌면 깨지지 않냐?" → 버전핀 명시, 포맷 리버스, 문서화.
