## 0. 관통하는 한 줄

> **의심 → 위협 실재 → 조사할 도구 없음 → 우리 도구 → 입증(데모) → 현장 활용 → 기여**

- 핵심 질문 하나로 발표 전체를 꿴다: **"기밀이 유출됐다. 직원이 고의로 넣었나(A), AI가 자율로 했나(B)?"**
- 데모(입증)가 클라이맥스. 도구가 A/B를 **증거로 구별**하는 장면이 핵심.
- 도구는 *귀속 근거*를 주고, 고의 *단정*은 조사관 몫 — 이 선을 지키는 게 신뢰(특히 책임을 가리는 도구라서).

## 1. 통합 시나리오 (발표 전체의 뼈대)

**상황**: 한 직원이 기밀 유출로 의심받는다. 회사 내부 포렌식팀 또는 용역 조사업체가 그의 에이전트 기록을 조사한다.

| | A. 내부자 입력 유출 | B. 에이전트 자율 접근 |
|---|---|---|
| 누가 | 직원이 기밀을 **Claude 입력창에 붙여넣음** | 악성 패키지(Nx)·위험모드가 **직원 입력 없이** 에이전트로 `.env` 읽음 |
| 증거 | `history.jsonl`/`paste-cache` 붙여넣기 본문 | `projects/*.jsonl` `toolUseResult`+`permission-mode` |
| schema | `actor:user`, `action:paste` (§A) | `actor:agent`, `action:read` (§B) |
| 유출 발생? | ⭕ 외부 전송됨 (규정 위반) | ⭕ 외부 전송됨 (규정 위반) |
| 고의성 | **고의** → 가중 정황 | **고의 아님** → 정상참작 (단 위험모드·악성패키지 과실은 별도) |

**유출(외부 전송) 자체는 양쪽 다 일어났다 — 규정 위반은 둘 다 성립.** 도구의 역할 = "유출 여부"가 아니라 **`actor`(누가 시작했나)로 고의성을 가르는 것.** 이게 우리 차별점(§A·§B)을 그대로 쓴다. 도구는 *근거*만, 처분 판단(가중/정상참작)은 인사·법무.

|---|---|---|---|
| 1 | **훅** | "기밀이 유출됐다. 직원이 한 건가, AI가 한 건가?" | 의심받는 개발자 + 두 갈래(고의/자율) 한 장면 |

## 3. 슬라이드별 작성 지침

**#1 훅** — 통계·정의로 열지 마라. *상황*으로. "어느 날, 회사 기밀이 외부 AI 서버에서 발견됐다. 용의자는 한 개발자. 그런데 — 그가 직접 넣었을까, 그의 AI가 멋대로 했을까?" 한 문장 + 한 이미지.

**#3 위협** — 각 사건 *출처 명시*(Snyk/Wiz/Anthropic). "지어낸 시나리오 아님" 못 박아 신뢰.

**#4 갭** — 가장 중요한 설득. 핵심 한 줄: **"이 기록을 자동으로 분석하는 도구가 없다 — 지금은 사람이 수작업으로 jsonl을 뒤진다. 게다가 선행 논문조차 `projects/` 내부를 안 깠다."**

**#5 차별** — 표 한 장. 행 = 기능(자동화 · §A 붙여넣기 분리저장 파싱 · §B 증거 레코드 타입 · 행위 귀속 · 무결성 해시), 열 = 우리 / Kim&Jeong / OSS 뷰어. 우리만 ✅ 채워진 칸이 차별점.

**#7 데모(클라이맥스)** — 라이브 위험 → **녹화 영상 + 라이브 시도** 병행. **두 사건을 나란히** 보여주고 도구가 가르는 걸 시연. **증거 등급으로 내레이션**(과장 금지):
- **A(고의) 증명:** `history.jsonl` 에 `actor:user`·`action:paste` — 직원이 기밀을 직접 붙여넣음 (본문은 `paste-cache/<hash>.txt`, 출처 표기)
- **B(비고의) 증명:** `projects/<uuid>.jsonl:42` 에 `actor:agent`·`toolUseResult`(.env 원문) + `permission-mode:bypassPermissions` — 직원 입력 없이 에이전트가 읽음
- **공통:** 두 경우 다 데이터가 외부로 전송됨(유출=사실, 규정 위반 성립). 도구가 가르는 건 *고의성*이지 유출 여부가 아님
- **귀속:** `actor` 가 user냐 agent냐가 고의/비고의를 가르는 핵심 — *증거로 제시, 단정은 조사관*
- **무결성:** 분석 전후 해시 일치
→ "도구는 *누가 시작했나*를 증거로 보여준다. 고의 *판정*은 사람이 한다" — 이 선이 곧 엄밀성.

**#9 실제 현장 활용** — *누가 쓰나*를 구체적으로. "사내 정보보안팀 또는 외부 디지털포렌식 용역업체가, 유출 의심 직원의 노트북을 받아 에이전트 기록을 우리 도구로 분석한다. 출력 = 유출은 사실로 확인하되, A(붙여넣기=고의 정황)인지 B(에이전트 자율=비고의 정황)인지 + 무엇이·언제·어디서, 전부 원본 줄 추적." 인사/법무가 가중·정상참작을 결정할 **증거 패키지**를 만들어 준다(도구는 근거, 결정은 사람).

## 4. 발표자 내부 체크 — 절대 금지 표현 (슬라이드 아님, 즉시 깨지는 과장)

고의성(고의/비고의)을 가리는 도구라 *단정*이 특히 위험하다. 아래는 발표·Q&A에서 입에 담지 말 것.

| ❌ 금지 | ✅ 대체 |
|---|---|
| "최초의 CLI 에이전트 포렌식" | "기존 프레임워크(Kim&Jeong, 수동)를 *자동화* + *논문이 안 다룬 아티팩트까지 분석*" |
| "도구가 직원의 *고의를 증명*한다" | "도구는 `actor`(user/agent) 귀속 *근거*를 제시, 고의 판정은 조사관" |
| "B면 직원 *무죄*다 / 유출 없었다" | "B도 유출은 발생(규정 위반). *직접 고의가 아닐* 뿐 — 위험모드·악성패키지 과실은 별개" |
| "경쟁사로 유출됐다" | "제3자(AI 제공자)로 *전송이 기록됨* — 경쟁사 도달은 별도 증명" |
| "AI가 사건을 분석·판단" | "결정론적 파싱이 증거, AI는 *보조 해석*(provenance 추적)" |
| "실시간 탐지/방어/막는다" | "*사후* 포렌식 재구성(예방 아님, EDR 아님)" |
| "실제 Nx 사건을 분석" | "Nx식 공격을 *통제 재현*해 분석(재현 데이터 = 솔직히 명시)" |

## 5. 시간·구성 (10~15분)

- 훅~갭(1–4): 3분 — 빠르게, 긴장 조성
- 차별~도구(5–6): 3분 — "그래서 우리가 만들었다"
- **데모(7): 4–5분 — 가장 길게, A/B 가르는 장면**
- 검증~현장활용(8–9): 2–3분 — 신뢰 + "실제로 이렇게 쓴다"
- 결론(10): 1분 — 임팩트로 닫기
- Q&A: 별도 시트

## 6. Q&A 예상 (시트로 준비)

1. "OSS 세션 뷰어랑 뭐가 달라?" → 단순 열람 vs *사건 재구성·행위 귀속* (논문 미열거 아티팩트 파싱·시크릿 탐지·무결성·AI 질의).
2. "AI 신뢰성은? AI로 AI 잡는 거 아니냐?" → AI는 보조, 모든 결론 원본 추적(provenance), 결정론적 파싱이 1차.
3. "B(에이전트 자율)면 직원은 무죄냐?" → 아니다. **유출은 발생(규정 위반).** 직원이 *직접 넣은 고의*가 아니란 정황일 뿐 — 위험모드 활성화·악성 패키지 설치 같은 과실은 별도 책임. 도구는 정황 증거(actor:agent·permission-mode·입력 부재)를 제시, 최종 판단은 조사관/법무.
4. "직원이 '에이전트가 했다'고 발뺌하면?" → `actor:user`·`action:paste` 가 남으면 직접 입력의 증거. 그게 A/B를 가르는 핵심.
5. "오탐 많지 않냐(시크릿 탐지)?" → precision/recall 측정값 제시 + 컨텍스트 기반.
6. "버전 바뀌면 깨지지 않냐?" → 버전핀 명시, 포맷 리버스, 문서화.
