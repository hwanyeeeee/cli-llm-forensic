## 0. 핵심 논리 (관통하는 한 줄)

> **문제 → 근거 → 갭 → 해법 → 입증 → 정직 → 기여**

- 데모(입증)가 **클라이맥스**. 모든 슬라이드가 데모를 향해 쌓이고, 데모 뒤는 신뢰(검증·정직)로 마무리.
- **정직(한계)이 신뢰를 만든다** = 교수님/심사 방어선. 과장 한 번에 전체가 깨짐.

## 1. 슬라이드 흐름 (10장 골격, 1슬라이드=1주장)

| # | 슬라이드 | 핵심 메시지 | 시각 자료 |
|---|---|---|---|
| 1 | **훅** | "당신의 AI 코딩 에이전트가 당신을 턴다" | Nx 사건 한 장면(악성 패키지→`claude --dangerously-skip-permissions`→시크릿 유출) |
| 2 | **배경** | CLI 코딩 에이전트 급증 + 능력(파일 접근·명령 실행·외부 연동) = 새 공격면 | 에이전트 아키텍처 도식 |
| 3 | **위협(근거)** | 위협은 *실재*한다 | 실사건 타임라인: Nx(2,180계정)·Replit(DB삭제)·Claude `.env` CVE·GTG-1002 |

| 6 | **도구 구조** | 2파트 + 7단계 파이프라인 | 수집→파싱→정규화→강화→복구→상관→리포트 + AI 질의 다이어그램 |
| 7 | **★데모** | *된다* — 사건이 재구성된다 | Nx식 재현 → 타임라인·시크릿탐지·귀속·무결성. **영상/GIF 백업 필수** |
| 8 | **검증** | 숫자로 증명 | precision/recall · 재구성등급(완전/부분/불가) · 무변경 해시 · 무료도구 대비 |

| 10 | **기여·결론** | 의의 + 기여 | 차별점 요약 · 학회 시의성(AI 포렌식 핫) · 2인 기여 매트릭스 |

## 2. 슬라이드별 작성 지침

**#1 훅** — 통계나 정의로 열지 마라. *충격적 사실*로. "개발자가 신뢰한 패키지가, 개발자의 AI한테, 개발자의 비밀을 훔치라고 시켰다." 한 문장 + 한 이미지.

**#3 위협** — 각 사건은 *출처 명시*(Snyk/Wiz/Anthropic). "이거 우리가 지어낸 시나리오 아님"을 못 박아 신뢰 확보.

**#4 갭** — 가장 중요한 설득. "왜 *포렌식 도구*인가". 핵심 한 줄: **"WSL2 안에서 벌어진 일을 Windows EVTX·EDR가 못 본다"** → 디스크 아티팩트 포렌식만이 답.

**#5 차별** — 표 한 장. 행 = 기능(자동화·Windows/WSL2·시크릿탐지·복구·무결성), 열 = 우리 / Kim&Jeong / OSS. 우리만 ✅ 채워진 칸이 차별점.

**#7 데모(클라이맥스)** — 라이브는 위험 → **녹화 영상 + 라이브 시도** 병행. **증거 등급으로 내레이션**(과장 금지):
- **아티팩트로 증명:** 08:35 `.env` Read 발생 (출처 `<uuid>.jsonl:42줄`)
- **추론:** actor=에이전트(자율 tool-call 패턴과 일치) — *증명 아닌 추론으로 명시*
- **로컬 캡처:** 시크릿 N개 inventory+base64 sink에 기록 (**로컬 sink** — "외부 도달"은 별도 증명, 절대 "외부로 유출됐다" 단언 금지)
- **무결성:** 분석 전후 해시 일치
→ 확신 경계를 *명시*할수록 포렌식 엄밀성이 드러나 오히려 강함.

**#9 정직** — 약점을 *먼저* 말하면 강점이 된다. "전송 성공은 증명 못 함(프록시 로그 병행 필요)" / "WSL2 EVTX 못 봄 — 이게 바로 우리 도구가 필요한 이유". 한계가 곧 존재이유로 연결.

## 3. 절대 금지 표현 (= 즉시 깨지는 과장)

| ❌ 금지 | ✅ 대체 |
|---|---|
| "최초의 CLI 에이전트 포렌식" | "기존 프레임워크(Kim&Jeong)를 *자동화·Windows/WSL2로 확장*" |
| "DLP가 못 본다 / 불가시" | "현 DLP는 브라우저·업로드 중심 → CLI는 *커버리지 갭*" |
| "AV가 못 잡는다" | (언급 X — AV는 무관) |
| "경쟁사로 유출됐다" | "제3자 제공자로 *전송이 로컬 캡처됨* — 경쟁사 도달은 별도 증명" |
| "AI가 사건을 분석·판단" | "결정론적 파싱이 증거, AI는 *보조 해석*(provenance 추적)" |

| "전송 성공 입증" | "*로컬 캡처·직렬화*까지 증명, 전송성공은 프록시/네트워크 로그 병행" |
| "실제 Nx 사건을 분석" | "Nx식 공격을 *통제 재현*해 분석(재현 데이터 = 강점, 솔직히)" |

## 4. 시간·구성 (10~15분 기준 예시)

- 훅~갭(1–4): 3분 — 빠르게, 긴장 조성
- 차별~도구(5–6): 3분 — "그래서 우리가 만들었다"
- **데모(7): 4–5분 — 가장 길게, 핵심**
- 검증~정직(8–9): 2–3분 — 신뢰
- 결론(10): 1분 — 임팩트로 닫기
- Q&A: 별도 시트 준비

## 5. Q&A 예상 (시트로 준비)

1. "OSS 세션 뷰어랑 뭐가 달라?" → 분석·복구·무결성·상관(열람만 vs 사건 재구성).
2. "AI 신뢰성은? AI로 AI 잡는 거 아니냐?" → AI는 보조, 모든 결론 원본 추적(provenance), 결정론적 파싱이 1차.
3. "WSL2에서 EVTX 못 보면 반쪽 아니냐?" → 그게 *문제 자체* — 우리는 디스크 아티팩트로 그 사각을 메움.
4. "오탐 많지 않냐(시크릿 탐지)?" → precision/recall 측정값 제시 + 컨텍스트 기반.
5. "이거 실제 사건에 썼냐?" → 통제 스테이징(ground-truth) + 실 머신 아티팩트 구조 검증.
6. "버전 바뀌면 깨지지 않냐?" → 버전핀 명시, 포맷 리버스, 한계로 문서화.
